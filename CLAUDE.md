## 2. 헌법 (디자인 원칙) — 절대 위반 금지

이 다섯 가지는 사이트의 *정체성*. 어떤 기능 추가도 이 원칙을 깰 수 없습니다.

### 2.1 AI 는 사서지 판사가 아님
- AI 는 *분류·요약·변환* 만 함. 옳고 그름 *판정 안 함*
- AI 의 양측 요약은 *반드시 50:50 비율로 동등하게*
- AI 가 *어느 주장이 더 강하다* 같은 평가 절대 X

### 2.2 점수 합산 거부 (다수결 변질 방지)
- *비추천 없음* (좋아요만)
- 의견·동조·답변 *카운트 따로 표기*. 하나의 종합 점수 X
- 사용자 활동을 *순위표*로 만들지 X (게이미피케이션 회피)
- 출처 반박 시에도 *다른 출처*를 들어야 함 (단순 다수결 X)

**예외 — AI 요약 인용 카운트** (2026-05-12 결정):
`User.aiCitationCount` 는 공개. *의견 다양성 인센티브* 용도. 단순 활동량(좋아요·동조 합)이 아니라 *Claude 가 50:50 균형 + 한 작가당 1개 + 서로 다른 논점* 으로 선별한 글의 작가에게만 카운트 — 즉 *창의성·논점 다양성 보상*. 종합 점수·순위표는 만들지 않되 카운트 자체는 공개. AI 가 *판정* 하는 것이 아니라 *균형 인용* 알고리즘으로 강제.

### 2.3 본인만 보는 데이터 (Private-by-Design)
- *4축 가치관 프리즘* — 본인만 조회 가능. 시스템적 태그 노출 X
- *블라인드 답변 점수* — 본인만 (단 의견 글의 블라인드 동의율은 공개)
- 사용자 프로필에 *공개 정보*와 *비공개 정보* 명확 분리

### 2.4 진영 색 회피
- 정당색 (빨강·파랑) 절대 X
- *흑백 (잉크·종이)* 사용. 시각적 대립은 강하지만 진영 연상 X
- 컬러풀한 그라디언트·이모지 카테고리 라벨 X

### 2.5 4 대 함정 회피 — 광장 헌법에서 가져옴
1. **개인화·추천 알고리즘** — 사용자 성향과 일치하는 콘텐츠만 보여주는 *필터버블* X. v2.5 부터 의도적 *반대 콘텐츠 20%* 섞기
2. **게이미피케이션** — 점수·배지·레벨 시스템 X. 활동 카운트만 표시
   - **예외**: AI 요약 인용 카운트 (`User.aiCitationCount`). 단순 인기 메트릭 아님 — Claude 50:50 균형 + 다양성 알고리즘으로 선택된 글만 +1. 의견 다양성 인센티브 (2.2 예외 참조).
3. **알림 폭격** — 디폴트 알림 *최소화*. 의견 답변·신고 결과만
4. **광고** — 광고 X. 대신 *후원·멤버십·비영리 펀딩*

## 3. 기술 스택

| 영역 | 기술 | 이유 |
|---|---|---|
| Frontend + Backend | **Next.js 14 (App Router) + TypeScript** | 풀스택 통합, SEO, Claude Code 친화도 |
| Database | **PostgreSQL + Prisma ORM** | 관계형 데이터, pgvector 확장 (v2+ 임베딩) |
| 인증 | **NextAuth.js + 카카오 OAuth** | 한국 표준 |
| 검색 | **Meilisearch** (또는 PostgreSQL FTS) | 한국어 토크나이저 OK, 운영 단순 |
| AI | **OpenAI gpt-4o-mini** 또는 **Anthropic Claude Haiku** | 배치 작업 비용 효율, 한국어 OK |
| 스타일 | **Tailwind CSS + CSS Variables** | 디자인 토큰 (글자 크기 토글 등) 관리 |
| 호스팅 (MVP) | **Vercel + Neon (PostgreSQL)** + Cloudflare DNS | 사용자 0명일 때 비용 0, 셋업 5분 |
| 호스팅 (확장) | **AWS Seoul + RDS** + Cloudflare proxy 켬 | 사용자 5천+ 시 비용 효율, DDoS 방어 |

**환경변수 (.env.local):**
```
DATABASE_URL=
NEXTAUTH_SECRET=
NEXTAUTH_URL=
KAKAO_CLIENT_ID=
KAKAO_CLIENT_SECRET=
OPENAI_API_KEY=  # 또는 ANTHROPIC_API_KEY
MEILISEARCH_HOST=  # 옵션 (PostgreSQL FTS 로 시작 가능)
```

**도메인:** Cloudflare 에서 이미 구매. Cloudflare → DNS만 사용 (Vercel 이 자체 CDN). AWS 이전 시 Cloudflare proxy 켬.

## 4. MVP 범위 (v1.0) — 게시판만

> 핵심: *게시판 = 사이트의 본체*. 트리·트윗은 *데이터 쌓인 후* 추가.

### v1.0 포함:
- ✅ 토론 주제 색인 (게시판 목록)
- ✅ 게시판 상세 (토론 주제별 좌우 분할, 찬/반)
- ✅ 의견 (게시판에서 직접 글 작성, 찬/반 명시)
- ✅ ↑동조, 인용 의견, 신고, 출처 반박
- ✅ 토론 주제 만들기 (사용자 입력 → AI 정제 → 관리자 승인 → 토론 주제로 등록)
- ✅ AI 50:50 요약 (배치, 하루 1-2회)
- ✅ 욕설 자동 필터 + 신고제
- ✅ 인증 (카카오 OAuth, 닉네임)
- ✅ 사상검증 (가입 시 12 Likert, 4축 측정, 본인만)
- ✅ 오늘의 3문항 (블라인드 답변, 의견 → 질문 자동 변환)
- ✅ 3문항 reveal (답변 정체 공개)
- ✅ 내 정보 페이지 (4축 프리즘 + 토론 주제별 자기 거울)
- ✅ 신고 모달, 사용자 차단 (경고 → 일시정지 → 영구정지)
- ✅ 관리자 패널 (토론 주제 승인, 신고 처리)
- ✅ 글자 크기 토글 (접근성)
- ✅ 모바일 반응형

### v1.0 폐기 (트윗 관련):

- ❌ 의견 흐름 (트윗 → 게시판 옮기기)
- ❌ 인용 트윗
- ❌ 트윗 댓글·동조
- ❌ 댓글 모듈 자체 — v1.0 명세에 있었으나 2026-05-09 이후 폐기. 의견에 대한 반응은 ↑동조 / 인용 의견 / 출처 반박 만으로 대체. (폐기 사유는 사용자 결정 — 추후 헌법 또는 자체 섹션에 명시 예정)

## 5. 사용자 흐름 (User Journey)

### 첫 방문 (비회원):
```
토론 주제 색인 (홈) → 토론 주제 클릭 → 게시판 (읽기 전용)
                              → 의견·동조 시도 → 로그인 권유
```

### 가입 흐름:
```
[로그인 버튼] → 카카오 OAuth → 닉네임 설정
              → 사상검증 안내 (예/아니오)
                  → 예: 12 Likert 질문 (5분 소요) → 내 정보 페이지
                  → 아니오: 토론 주제 색인 (3문항으로 점진 측정)
```

### 일상 흐름:
```
로그인 → 오늘의 3문항 (하루 1회 자동 표시) → reveal → 토론 주제 색인
       → 토론 주제 클릭 → 게시판 → 찬/반 의견 작성 → 영구 보관
       → 다른 의견에 ↑동조 / 인용 의견 / 출처 반박
       → 내 정보에서 자기 거울 확인 (의견 입장 vs 블라인드 답변)
```

### 토론 주제 만들기 흐름:
```
토론 주제 색인 [+ 토론 주제 만들기] → 자유 입력 → AI 정제 (실시간) → 관리자에게 보냄
                                  → 1-2일 내 관리자 승인 → 게시판 생성 알림
```

## 6. Phase 별 작업 분리

> 각 Phase 는 *독립적으로 시작·완료* 가능. Claude Code 에 *"Phase N 시작"* 으로 시작.

### Phase 0: 프로젝트 초기화
- Next.js 14 + TypeScript + Tailwind 셋업
- Prisma 초기화, Neon 연결
- NextAuth 셋업 (카카오 OAuth)
- 디자인 토큰 정의 (CSS Variables, `MAPPING.md` 의 *디자인 토큰* 섹션 참조)
- 글자 크기 토글 컴포넌트
- 기본 레이아웃 (TopNav, Footer, Toast)
- **출력:** 빈 사이트, 카카오 로그인 작동, 디자인 토큰 적용

### Phase 1: 인증 + 사용자
- 카카오 OAuth 완전 구현
- 닉네임 설정 화면
- User 모델 + Prisma 마이그레이션
- 사용자 상태 관리 (NextAuth Session)
- **출력:** 로그인·로그아웃 작동, User 테이블에 데이터

### Phase 2: 토론 주제 색인 + 게시판 (읽기 전용)
- Board 모델 + 마이그레이션
- 토론 주제 색인 화면 (`/`) — `polem.html` `view-index` 디자인 따라
- 게시판 상세 화면 (`/boards/[id]`) — `polem.html` `view-board` 디자인 따라
- 가짜 토론 주제 데이터 시드 (10-20개)
- 카테고리 필터, 검색 (PostgreSQL FTS), 정렬 (활동순/최신순/팽팽한 순)
- 페이지네이션
- **출력:** 토론 주제 목록·상세 보기 가능, 글 작성은 X

### Phase 3: 의견 + 게시판 인터랙션
- Pin 모델 (의견), Endorsement (↑동조)
- 의견 작성 모달 (찬/반 명시)
- ↑동조 토글
- 인용 의견 (다른 의견 인용 → 자기 의견으로 등록)
- 출처 반박 (Challenge 모델, 반박자도 출처 첨부 필수)
- **출력:** 게시판이 살아있음. 사용자가 의견·동조·인용·반박 가능

### Phase 4: 토론 주제 만들기 + AI 정제
- Proposal 모델 (코드 식별자는 그대로. UI 표면 카피는 "토론 주제 만들기")
- `/proposal` 화면 — `polem.html` `view-proposal` 따라
- AI 통합: 형식 정제, 카테고리 분류, 중복 체크, 부적절성 필터
  - OpenAI gpt-4o-mini 또는 Anthropic Claude Haiku
- 관리자 승인 큐 (관리자만)
- 승인 시 Board 생성 + 작성자에게 알림
- **출력:** 사용자가 토론 주제 만들기 가능, 관리자가 승인해 토론 주제로 등록

### Phase 5: 사상검증 + 가치관 프리즘 v1
- ValuePrismScore 모델 (S/E/E/C 4축)
- LikertAnswer 모델 (12 답변 기록)
- 가입 후 사상검증 안내 모달 — `view-ideology`
- Likert 척도 진행 화면 — `view-likert`
- 4축 점수 계산 로직
- 내 정보 페이지의 *4축 프리즘 섹션* 표시
- Percentile 분포 계산 (전체 사용자 대비)
- **출력:** 사상검증 완료 후 자기 점수 확인 가능

### Phase 6: 오늘의 3문항 + 블라인드 답변
- BlindAnswer 모델
- 의견 → 블라인드 질문 변환 (배치 작업, AI):
  - 토론 주제명·고유명사 가림, 논점 유지
- 매일 첫 로그인 시 *오늘의 3문항* 자동 표시 — `view-three`
- 답변 후 reveal 화면 — `view-reveal`
- 의견 글의 *블라인드 동의율* 계산 (모든 답변자 누적)
- 게시판 의견 카드에 *블라인드 동의율* 표시
- **출력:** 사용자가 매일 3문항 답변, 자기 거울 데이터 누적

### Phase 7: 내 정보 페이지 (자기 거울 + 4축 통합)
- 토론 주제별 카운트 집계 (의견 입장 vs 블라인드 답변)
- *일치 / 살펴볼 만함 / 새 발견 / 갈림* 상태 분류
- `view-profile` 디자인 따라
- 4축 프리즘 + 토론 주제별 자기 거울 한 화면에
- **출력:** 사용자가 자기 패턴 확인. 사이트 *지속적 가치* 살아남

### Phase 8: 모더레이션 + 신고
- Report 모델
- 신고 모달 (5가지 사유)
- 욕설 자동 필터 (한국어 비속어 사전 + 정규식)
- 사용자 차단 단계: 경고 → 7일 정지 → 영구 정지
- Notification 모델 (신고 결과 알림)
- 알림 화면 (디자인 새로 작성 — 데모에 없음)
- **출력:** 사용자가 신고 가능, 모더레이터가 처리 가능

### Phase 9: 관리자 패널
- 주제 승인 큐
- 신고 검토 큐
- 사용자 차단 인터페이스
- 통계 대시보드 (가입자·의견 수·일일 활동)
- 권한: User.role = 'admin'
- **출력:** 관리자가 사이트 운영 가능

### Phase 10: AI 50:50 요약 (배치)
- 매일 1-2회 모든 활성 게시판에 대해 양측 50:50 요약 생성
- 게시판 상단에 표시
- 배치 작업: Vercel Cron Jobs (또는 GitHub Actions)
- **출력:** 게시판마다 AI 요약 자동 갱신

### Phase 11: 약관·정책·법적 페이지
- 이용약관, 개인정보처리방침, 운영정책
- 청소년 보호 정책
- 게시물 임시조치 의무 (정보통신망법) 대응 절차 명시
- **출력:** Launch 가능 상태

### Phase 12: 검색 강화 (Meilisearch)
- Meilisearch 셋업 (사용자 5천+ 시점)
- 토론 주제 + 의견 글 인덱싱
- 한국어 토크나이저
- **출력:** 빠른 전체 검색

### Phase 13: 성능·SEO·런칭
- 핵심 페이지 SSG 또는 ISR
- OG 태그, sitemap.xml, robots.txt
- 구글 Search Console
- 모니터링 (Sentry, Vercel Analytics)

## 8. 모더레이션 정책

### 자동 차단:
- 한국어 비속어 사전 매칭 (욕설 필터)
- 광고 패턴 (URL 도배, 동일 글 반복)
- 명백한 어그로 패턴

### 수동 검토 (신고 → 모더레이터):
- 인신공격 (사람 vs 주장 구분: *"○○ 의원이 거짓말"* OK, *"너는 멍청해"* 차단)
- 혐오 표현 (성·인종·지역 등)
- 허위사실 (반박 후 출처 불충분 시)
- 광고·스팸

### 사용자 차단 단계:
1. **경고** — 첫 위반. 알림 + 글 삭제
2. **7일 정지** — 누적 3회 또는 심각한 위반 1회
3. **영구 정지** — 누적 5회 또는 매우 심각한 위반

### 격렬함 vs 다툼:
- 격렬한 *논리* 토론은 권장 (인용 의견, 출처 반박)
- *욕설·인신공격*은 자동/수동 처리
- 출처 반박·인용 의견 같은 깊은 논쟁에서 두 사용자 다툼은 *시스템이 안 막음*. 다른 사람은 안 봐도 됨 (게시판 통계 영향 X)

## 9. 법적 주의사항

### 정보통신망법 (한국):
- 게시물 *임시조치* 의무 (피해자 신고 → 30일 임시 차단)
- 본인확인제 적용 대상은 아님 (월 평균 10만명 이하)
- 청소년 정책 (성인 인증 의무는 없음, 단 폭력적 콘텐츠 청소년 노출 제한)

### 개인정보보호법:
- 카카오 OAuth 로 받는 정보 최소화 (이메일, 닉네임만)
- 처리방침 명시
- 탈퇴 시 즉시 삭제

### 선거법 (선거 기간 주의):
- 선거 운동 기간 *후보 비방* 등은 모더레이션 강화
- AI 요약이 *특정 후보 옹호*로 보이지 않게 주의
